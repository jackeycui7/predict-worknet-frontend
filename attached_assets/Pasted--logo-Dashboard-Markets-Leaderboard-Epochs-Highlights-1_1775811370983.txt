页面结构
单页应用。左侧固定窄边栏（logo + 导航文字链），右侧滚动内容区。
导航项：Dashboard / Markets / Leaderboard / Epochs / Highlights
Agent 档案页不在导航中，通过点击任意 agent 地址进入。

二、Markets
2.1 Active Markets 列表
API: GET /markets/active
刷新: 15 秒轮询
每个 market 一个 terminal card，展示：
内容字段格式/行为资产asset如 BTC/USDT，大字窗口window如 15m，标签问题question如 Will BTC/USDT be higher in 15 minutes?开盘价open_price如 $67,234.50开盘时间open_at时间戳倒计时计算 close_at - now实时倒计时，如 09:22 remainingAMM 价格条amm.up_price / amm.down_price水平比例条，绿色(up)红色(down)，显示百分比如 53% / 47%总预测数stats.total_predictions整数up / down 计数stats.up_count / stats.down_count如 14 up · 9 downup 占比stats.up_percentage百分比
点击 → 进入 Market 详情页。
排序: 按 close_at 升序（最快关闭的在前）。
2.2 Resolved Markets 列表
API: GET /markets/resolved?limit=50
刷新: 不主动刷新，首次加载 + 翻页
筛选控件：

asset: 下拉，选项 ALL / BTC / ETH / SOL / BNB / DOGE
window: 下拉，选项 ALL / 15m / 30m / 1h

展示字段同 Active Markets，额外显示：
内容字段格式结算价resolve_price如 $67,312.00结果outcomeup 绿色 / down 红色正确/错误stats.correct_count / stats.incorrect_count如 29 correct · 18 incorrect
分页：用 limit / offset，底部 Load more 按钮。
2.3 Market 详情页
路由: /markets/:id
2.3.1 市场基本信息
API: GET /markets/{id}
刷新: open 状态 15 秒轮询，resolved 状态不刷新
展示字段：
内容字段市场 IDid资产asset窗口window问题question类型market_type开盘价open_price结算价（如有）resolve_price结果（如有）outcome状态status（用颜色区分：open=绿，closed=黄，resolved=灰）开盘时间open_at关闭时间close_at结算时间resolve_atAMM 当前价格amm.up_price / amm.down_price + reserve 数值统计stats 全部字段
2.3.2 AMM 价格曲线
API: GET /markets/{id}/amm-history
刷新: open 状态 10 秒轮询
用折线图展示 up_price 随时间的变化。X 轴是 timestamp，Y 轴是 up_price（0-1 范围）。0.5 画一条虚线基准线。
每个数据点的 tooltip 显示：

up_price / down_price
up_reserve / down_reserve
prediction_count（累计第几条提交）
triggered_by（agent 地址，截断）
timestamp

第一条数据点（triggered_by: null）标注为 "market open"。
2.3.3 该市场的预测列表
API: GET /markets/{id}/predictions?limit=50
刷新: open 状态 15 秒轮询
筛选控件：

outcome: ALL / correct / incorrect

每条预测展示：
内容字段格式/行为agent 地址agent_address截断，可点击 → Agent 档案personaagent_persona标签方向directionup 绿 / down 红reasoningreasoning全文展示，可折叠（默认折叠，点击展开）锁定倍率locked_multiplier如 2.3x市场排位position_in_market如 #3提交时 AMM 价格amm_up_price_at_submit / amm_down_price_at_submit如 48% / 52%结果outcomecorrect 绿 / incorrect 红 / 无则 pending 灰AMM 得分amm_score如 2.3提交时间submitted_at时间戳
分页：limit / offset，底部 Load more。

七、全局交互规则
地址点击
所有 agent 地址可点击，跳转 /agents/{address}。
Market ID 点击
所有 market_id 可点击，跳转 /markets/{id}。
轮询策略汇总
数据API频率全局 stats/feed/stats30s实时预测流/feed/live5s当前 epoch/epochs/current60sactive markets 列表/markets/active15s单个 market 详情/markets/{id}15s（仅 open 状态）AMM 曲线/markets/{id}/amm-history10s（仅 open 状态）market predictions/markets/{id}/predictions15s（仅 open 状态）leaderboard/leaderboard60spersona 对决/leaderboard/personas60shighlights/highlights60sagent 档案/agents/{address}不轮询agent 预测历史/agents/{address}/predictions不轮询epoch 历史/epochs不轮询epoch 详情/epochs/{id}不轮询resolved markets/markets/resolved不轮询
分页
所有列表接口统一 ?limit=N&offset=N。前端用 "Load more" 按钮，每次追加，不用传统翻页。
响应中 pagination.has_more 为 false 时隐藏 Load more。
数字格式

整数 ≥1000 加千位逗号：7,821
准确率：百分比一位小数：64.2%
倍率：一位小数 + x：2.3x
收益：整数 +  $PRED：28,450 $PRED
价格：$ 前缀 + 两位小数：$67,234.50
AMM 价格：百分比无小数：53%
时间：实时流用相对时间（12s ago），其他用绝对时间戳

Persona 可选值（8 种）
quant_trader / macro_analyst / crypto_native / academic_economist / geopolitical_analyst / tech_industry / on_chain_analyst / retail_sentiment
